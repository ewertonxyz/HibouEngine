# HibouEngine

A game engine built as a study project to demonstrate proficiency in **C++/C# interoperability**, **custom build tooling**, and **real-time 3D rendering**.

The engine runs as a native C++ DLL (`Engine.dll`) while the editor is a C# WPF application that hosts the rendering surface via P/Invoke. Communication between the two layers happens through an `extern "C"` ABI, with the editor passing an `HWND` to the engine at initialization time.

---

## Architecture

```
HibouEngine/
├── Engine/       C++ DLL — ECS, OpenGL renderer, GLTF loader, interop exports
└── Editor/       C# WPF application — hosts the viewport, project/level management
```

### Engine (C++)

| Area | Details |
|---|---|
| **ECS** | [EnTT](https://github.com/skypjack/entt) registry. `Scene` owns the registry; `Entity` is a thin wrapper. Components are plain structs (`TransformComponent`, `MeshComponent`, `CameraComponent`, `DirectionalLightComponent`, etc.). |
| **Renderer** | `IGraphicsAPI` interface with an `OpenGLManager` implementation. `Renderer` iterates ECS views and issues draw calls using a Cook-Torrance PBR shader. |
| **Model loading** | `ModelLoader` loads GLTF via [TinyGLTF](https://github.com/syoyo/tinygltf). Meshes are uploaded to interleaved VAOs (position + normal + UV). |
| **Interop exports** | `extern "C"` functions in `EngineInterop.cpp`: `InitializeGameEngine`, `RenderEngineFrame`, `ResizeEngineViewport`, `ShutdownGameEngine`. |
| **Build** | [Sharpmake](https://github.com/ubisoft/Sharpmake) generates the Visual Studio solution and project files from `HibouEngine.sharpmake.cs`. Dependencies managed via [vcpkg](https://vcpkg.io). |

### Editor (C#)

| Area | Details |
|---|---|
| **Viewport** | `OpenGLViewport` (inherits `HwndHost`) creates a child Win32 window and passes its `HWND` to the engine. WPF's `CompositionTarget.Rendering` drives the render loop. |
| **Project format** | Custom `.hxproj` (JSON) and `.hxlevel` (JSON) files. Loaded by `ProjectLoader` and `LevelLoader`; created by `ProjectCreator`. |
| **Scene explorer** | MVVM hierarchy: `LevelLoader` → `HxLevel` → `SceneViewModel` → `ExplorerView`. |
| **Asset browser** | `AssetBrowserView` + `AssetBrowserViewModel` — recursive directory tree with thumbnail previews, inline rename, and search. |
| **Node graph** | UI-only node graph editor using [Nodify](https://github.com/miroiu/nodify) (backend not yet wired). |
| **MVVM** | [CommunityToolkit.MVVM](https://learn.microsoft.com/en-us/dotnet/communitytoolkit/mvvm/) for observable properties and commands. |

## Key Highlights

- **Native/managed interop**: The C++ engine DLL is loaded at runtime by the C# editor (`NativeLibrary.Load`). All cross-boundary calls go through a flat `extern "C"` API declared in `EngineBindings.cs` via P/Invoke.
- **Custom build system**: Project files are generated by Sharpmake (C# DSL), not maintained by hand. Run `GenerateProjects.bat` to regenerate after changing `HibouEngine.sharpmake.cs`.
- **PBR shading**: Cook-Torrance BRDF (GGX distribution, Smith geometry, Schlick-Fresnel) with albedo, normal, metallic/roughness, and occlusion texture support. Screen-space TBN computation requires no tangent vertex attribute.
- **Custom project format**: `.hxproj` and `.hxlevel` JSON files describe project settings and scene entities independently of any third-party format.
- **ECS-driven scene**: All scene state lives in an EnTT registry. The renderer, level loader, and editor all interact with the same component types.

## Building

### Prerequisites

- Visual Studio 2026 with **Desktop development with C++** and **.NET desktop development** workloads
- vcpkg integrated with Visual Studio (`vcpkg integrate install`)
- .NET 10 SDK

### Steps

1. **Generate project files**

   ```bat
   GenerateProjects.bat
   ```

   This runs Sharpmake and regenerates `hibouengine.sln` and `Engine/engine.vcxproj`.

2. **Open the solution**

   Open `hibouengine.sln` in Visual Studio 2026.

3. **Build**

   Build the solution (`Ctrl+Shift+B`). The engine DLL is output to the Editor's binary directory so it is available at runtime.

4. **Run**

   Set **Editor** as the startup project and press F5, or run:

   ```bat
   run.bat
   ```

> **Note:** The shader paths in `Renderer.cpp` and asset icon paths in `AssetBrowserViewModel.cs` currently contain absolute paths and will need to be updated to match your local setup.

```
│   ├── Components/           UI panels (AssetBrowser, Explorer, Console, etc.)

├── Engine/
│   ├── include/              Public headers (ECS, Graphics, Interop)
│   ├── src/                  Implementation files
│   └── vendor/               EASTL, EABase, stb, TinyEXR submodules
├── Tools/
│   └── Sharpmake/            Build tool submodule
├── HibouEngine.sharpmake.cs  Build configuration
└── GenerateProjects.bat      Regenerates VS solution and project files
```
